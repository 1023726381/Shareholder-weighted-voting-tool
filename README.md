## 系统概述

本系统分为**桌面管理端**（管理员使用）和**移动网页端**（投票人使用）。支持两种投票模式：
- **百分比投票模式**：投票人将自己的持股数分配给候选人（整数股），可添加额外候选人，并可配置是否允许弃票（非零人数限制）。
- **一人一票模式**：投票人选择指定数量的候选人，可添加额外候选人。

系统还提供现场签到、实时日志、排行榜导出等功能。

## 管理员快速上手

### 2.1 启动程序

双击运行 `vote_system.exe`（或运行 `python vote_system.py`），弹出管理主窗口。

### 2.2 配置投票项目

左侧面板“投票基础配置”：
- **投票标题**：输入标题，点击“更新标题”。
- **候选人列表**：  
  - 点击“批量添加”弹出窗口，每行输入一个候选人姓名（可包含中英文），保存。  
  - 点击“删除选中”或“清空”进行删除操作。
- **投票模式**：选择“百分比投票”或“一人一票”。
- **允许弃票**：  
  - 百分比模式下：勾选后，分配股数总和可以少于持股数（弃票）；不勾选时必须恰好分配全部持股数。  
  - 一人一票模式下：勾选后，选择人数可以少于当选人数；不勾选时必须恰好选择当选人数。
- **允许添加额外候选人**：勾选后，投票人在页面可自行添加额外候选人。

### 2.3 管理选举人

右侧面板“选举人管理与日志”：
- **选举人列表**：显示所有有投票资格的人员（格式：姓名+身份证号）。  
- **批量添加**：点击后弹出窗口，每行输入“姓名+身份证号”（例如“张三11010119900307663X”），点击保存。  
- **删除选中**：从列表中删除选中的选举人。  
- **复制全部**：将当前选举人列表复制到剪贴板。  
- **清空**：删除所有选举人。  
- **导入签到用户**：将已签到但尚未在选举人列表中的用户自动添加进去。  
- **查看签到池**：显示所有已签到人员（身份证号脱敏），支持自动滚动和字体缩放。

**提示**：选举人列表中的名称必须与“持股明细文件.xlsx”中的股东名称完全一致（包括大小写、空格、身份证号字母大写）。

### 2.4 启动服务器与生成二维码

中间面板“服务器与数据操作”：
- **启动服务器**：  
  1. 点击后弹出“设置有效票最低分数”（百分比模式需要，一人一票模式仅设置当选人数）。  
  2. 设置“当选人数”（百分比模式也需设置，用于限制非零人数和排名）。  
  3. 服务器启动后，地址框显示 `http://本机IP:5000`，同网络下的手机可访问。
- **停止服务器**：结束投票服务。
- **签到QR码**：生成签到页面二维码，供投票人扫码签到。  
- **服务器地址QR码**：生成首页二维码，可直接进入投票首页。  
- **计算投票结果**：生成排行榜并导出Excel文件（保存到当前目录）。  
- **复位投票数据**：清空所有投票记录（保留候选人/选举人配置），用于第二次投票。

### 2.5 监控投票进度

右侧面板“实时投票记录”区域：
- **显示/隐藏日志**：切换日志详细程度。  
- **暂停日志**：停止自动刷新，再次点击恢复。  
- **复制日志**：将当前日志内容复制到剪贴板。  
- 日志内容：百分比模式下显示每位投票人给每个候选人分配的股数；一人一票模式下显示选中/未选状态。

右侧“未投票人员”列表会实时更新，方便管理员跟踪。

### 2.6 签到管理

- 生成“签到QR码”，投票人扫码后进入 `/signin` 页面，输入姓名和身份证号完成签到。  
- 管理员在“选举人管理与日志”区域点击“查看签到池”，可看到所有已签到人员（身份证号脱敏）。  
- 点击“导入签到用户”，系统将签到记录中尚未在选举人列表的人员自动添加，省去手动录入。

### 2.7 结束投票并计算结果

1. 确认所有人都已投票后，点击“停止服务器”。  
2. 点击“计算投票结果”，系统会：
   - 百分比模式：读取持股明细文件，计算每位候选人的总得票数（加权），与基准线比较，确定是否有效票，并按照当选人数标记“当选”状态。  
   - 一人一票模式：直接统计票数，按得票数排序并标记当选。  
3. 生成排行榜窗口，可导出Excel文件（保存在程序所在目录）。

## 投票人操作指南

### 3.1 签到（如果需要）

- 使用手机扫描管理员提供的“签到QR码”，进入签到页面。  
- 输入“姓名”和“身份证号”，点击“签到”。  
- 签到成功后页面会提示“签到成功”，然后可进入投票。

### 3.2 投票流程（百分比模式）

1. **进入投票页面**  
   - 扫描“服务器地址QR码”或直接访问 `http://服务器IP:5000`。  
   - 在首页输入“姓名+身份证号”（例如“张三11010119900307663X”），点击“进入投票”。  
2. **查看规则**  
   - 页面顶部显示您的持股数、已分配股数、剩余股数。  
   - 规则说明：必须将您的持股数分配给候选人（整数股），允许弃票时总和可以少于持股数，否则必须等于持股数；最多允许若干位候选人获得非零股数。  
3. **分配股数**  
   - 每个默认候选人后面有一个数字输入框，直接输入整数股数（不能超过您的持股数）。  
   - 如需添加额外候选人，点击“+ 添加额外候选人”，输入姓名和股数，点击“确认”。确认后该候选人变为只读，并显示“删除”按钮。  
   - 可以随时修改默认候选人的股数，系统会自动计算已分配总和和剩余股数。  
4. **提交投票**  
   - 当所有校验通过（股数总和符合规则、非零人数未超限），提交按钮变为绿色，点击“提交投票”。  
   - 确认提交后，页面会显示“投票成功”，并尝试自动关闭（微信环境下效果最佳）。  
   - 如果窗口未自动关闭，可手动关闭。

### 3.3 投票流程（一人一票模式）

1. **进入投票页面**（同上）。  
2. **勾选候选人**  
   - 直接勾选默认候选人前的复选框。  
   - 如需添加额外候选人，点击“+ 添加额外候选人”，输入姓名，确认后会出现复选框，可勾选。  
3. **提交投票**  
   - 当选人数限制会显示在规则中，需勾选恰好（或最多，取决于“允许弃票”）指定人数。  
   - 校验通过后提交。

### 3.4 常见问题与提示

- **登录失败**：请确认输入的“姓名+身份证号”与管理员添加的选举人列表完全一致（注意身份证号字母大写，无空格）。  
- **重复投票**：每个身份证号只能投票一次。  
- **股数分配错误**：如果剩余股数为负数，或非零人数超过限制，提交按钮会禁用并显示红色提示。  
- **额外候选人**：添加额外候选人时必须输入姓名和股数（百分比模式）或姓名（一人一票模式），且不能与默认候选人重复。  
- **微信中关闭窗口**：投票成功后系统会自动调用微信关闭接口，如未关闭，请手动关闭页面。

## 系统维护

### 4.1 数据备份与复位

- **投票数据**：保存在 `vote_backup.json`，可定期复制备份。  
- **签到数据**：保存在 `signin_records.json`。  
- **复位投票数据**：在管理窗口点击“复位投票数据”，可清空所有投票记录（候选人/选举人配置保留），以便重新投票。
